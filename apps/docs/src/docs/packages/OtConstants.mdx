# `@ot/constants`

Application-wide constants and generated GraphQL types for the Open Targets Platform.

**Package name:** `@ot/constants`  
**Location:** `packages/ot-constants/`  
**Entry point:** `packages/ot-constants/src/index.ts`

---

## Two categories of content

### 2. Generated GraphQL types (`src/types/index.ts`)

This file (~140 KB) is the output of `graphql-codegen` against the production Open Targets GraphQL schema. **Do not edit it by hand** — regenerate with:

```bash
yarn generateAPITypes
```

It exports:
- All GraphQL scalars mapped to TypeScript primitives
- Every object type in the schema (e.g. `Target`, `Disease`, `Drug`, `Variant`, `Study`, `CredibleSet`, `Evidence`, `Interaction`, `MousePhenotype`, …)
- All query/mutation/fragment operation types
- `GraphQLClient` and `RequestOptions` types from `graphql-request`

#### Key type exports

```ts
// Scalar types
export type Maybe<T> = T | null;
export type Scalars = { ID, String, Boolean, Int, Float, Long }

// Entity types (sample)
export type Target = { /* id, approvedSymbol, biotype, ... */ }
export type Disease = { /* id, name, description, ... */ }
export type Drug = { /* id, name, maximumClinicalTrialPhase, ... */ }
export type Variant = { /* variantId, chromosome, position, ... */ }
export type Study = { /* studyId, studyType, nSamples, ... */ }
export type CredibleSet = { /* credibleSetId, pValueMantissa, ... */ }

// Association types
export type AssociatedTarget = { /* score, datatypeScores, target: Target */ }
export type AssociatedDisease = { /* score, datatypeScores, disease: Disease */ }

// Evidence types
export type GeneticAssociationEvidence = { /* ... */ }
export type SomaticMutationEvidence = { /* ... */ }
// ... one type per data source
```

#### Usage in components

```ts
import type { Target, AssociatedDisease } from "@ot/constants";

function TargetCard({ target }: { target: Target }) {
  return <div>{target.approvedSymbol}</div>;
}
```

For query result types, import the generated operation type alongside the query document:

```ts
import type { TargetAssociationsQuery } from "@ot/constants";
import TARGET_ASSOCIATIONS_QUERY from "./TargetAssociationsQuery.gql";

const { data } = useQuery<TargetAssociationsQuery>(TARGET_ASSOCIATIONS_QUERY);
```

## Data types reference

The platform groups evidence into data types, each with an `id` and display `label`:

| ID | Label |
|---|---|
| `genetic_association` | Genetic associations |
| `somatic_mutation` | Somatic mutations |
| `clinical` | Clinical |
| `affected_pathway` | Pathways & systems biology |
| `literature` | Text mining |
| `rna_expression` | RNA expression |
| `animal_model` | Animal models |
| `ot_partner` | Partner data _(partner preview only)_ |
