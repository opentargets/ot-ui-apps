# Getting Started

## Prerequisites

- Node.js 20+
- Yarn 1.22.x (`packageManager` is locked in `package.json`)
- The repo cloned from [github.com/opentargets/ot-ui-apps](https://github.com/opentargets/ot-ui-apps)

## Install and run docs

```bash
# From the repo root — installs all workspaces
yarn install

# Start the docs dev server (port 6006)
yarn dev:docs

# Build the static site
yarn build:docs
```

## Run the full platform app

```bash
yarn dev:platform
```

## Useful root scripts

| Script | What it does |
|---|---|
| `yarn dev:docs` | Docs dev server (Storybook, port 6006) |
| `yarn dev:platform` | Platform app dev server |
| `yarn build:docs` | Static Storybook build → `apps/docs/storybook-static/` |
| `yarn build:platform` | Platform production build |
| `yarn lint` | Biome check across all workspaces |
| `yarn lint:fix` | Biome autofix |
| `yarn check:fix` | Biome format + lint fix |

## Branch conventions

Active feature branches use the prefix `cc-` (Carlos Cruz). The main integration branch is `main`. The `cc-story` branch carries the Storybook setup and is kept rebased on `main`.

## Adding a UI component story

Stories live **in the package they document** — never copy components into `apps/docs`.

1. Create a `MyComponent.stories.tsx` next to the component source:

```tsx
// packages/ui/src/components/MyComponent/MyComponent.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import MyComponent from "./MyComponent";

const meta: Meta<typeof MyComponent> = {
  title: "UI Components/MyComponent",
  component: MyComponent,
  parameters: { layout: "centered" },
  tags: ["autodocs"],   // ← opt-in: generates a Docs page for this component
};

export default meta;
type Story = StoryObj<typeof MyComponent>;

export const Default: Story = {
  args: { /* realistic props */ },
};
```

2. Storybook hot-reloads automatically — no config change needed.

## Adding a sections story

Sections are complex, data-fetching widgets. Use the `OTApolloProvider` decorator (already wired in `preview.ts`) and pass static fixture data via `args` or `loaders`.

```tsx
// packages/sections/src/target/MySection/MySection.stories.tsx
import type { Meta, StoryObj } from "@storybook/react";
import { MySection } from "./MySection";

const meta: Meta<typeof MySection> = {
  title: "Sections/Target/MySection",
  component: MySection,
  tags: ["autodocs"],
};
export default meta;
```

The dev server points `window.configUrlApi` at the **production GraphQL endpoint**, so stories can make real API calls during development.

## Adding a narrative docs page

Drop an `.mdx` file under `apps/docs/src/docs/`. Use `<Meta title="...">` to position it in the sidebar:

```mdx
import { Meta } from "@storybook/addon-docs/blocks";

<Meta title="My Section/My Page" />

# My Page
Content here.
```

## Formatting

All files use **Biome** (not ESLint/Prettier). Run `yarn check:fix` before committing. A Husky pre-commit hook enforces this.

## TypeScript

Strict mode is enabled in all packages. `noUnusedLocals`, `noUnusedParameters`, and `verbatimModuleSyntax` are all on. Do not use `// @ts-ignore` — fix the types.
